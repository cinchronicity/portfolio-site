Project Overview

The FlixHive Angular App is the client-side of a full-stack movie application built with Angular and Angular Material. It connects to the existing FlixHive API to let users browse movies, view detailed information about genres and directors, and manage their personal profiles and favorite movies. The project highlights modern front-end development practices, component-based architecture, and responsive UI design using Angular‚Äôs robust framework.

Goal / Purpose

The objective was to master Angular framework fundamentals and advanced concepts including TypeScript, reactive programming, and Material Design implementation. This project emphasized creating a professional-grade single page application that consumes a RESTful API. It also focused on developing clean, well-documented, and scalable code while practicing Agile methodologies and collaboration skills essential in real-world development environments.

Challenges & Solutions

Challenge: Frontend successfully authenticated locally but failed in production due to CORS policy blocking requests between GitHub Pages and Heroku backend.

Solution: Modified backend API to allow all origins for portfolio demonstration. Balanced security considerations with accessibility for potential employers to ensure seamless functionality across different deployment environments.

Challenge: Application failed to deploy to GitHub Pages due to Angular's updated build output structure and incorrect base href configuration.

Solution: Updated base href from placeholder values to actual GitHub Pages URL. Modified deployment path to accommodate Angular v17+'s new /browser subdirectory structure. Corrected deployment command: npx angular-cli-ghpages --dir=dist/flix-hive-angular-client/browser

Challenge:Peer dependency conflicts when installing development tools due to minor version mismatches between Angular packages.

Solution:Used --legacy-peer-deps flag for non-critical development dependencies. Maintained stable core Angular ecosystem while enabling additional tooling.

Lessons Learned

  • Employed systematic debugging techniques to identify and resolve issues efficiently
  • Adapted to Angular v17+ build structure changes and broke down and solved complex deployment challenges by isolating and addressing multiple technical layers
  • Managed dependencies strategically to maintain Angular ecosystem stability while applying targeted fixes
  • Integrated secure JWT authentication, CORS policies, and RESTful API communication between the Angular frontend and Node.js backend
  • Enhanced project professionalism with thorough TypeDoc documentation, responsive Angular Material design, and a production-ready deployment pipeline
