Project Overview

The FlixHive API is the server-side component of a full-stack movie web application built with Node.js, Express, and MongoDB. It provides secure endpoints that deliver movie, director, actor, and genre data in JSON format, while allowing users to register, update their profiles, and manage their list of favorite movies. This RESTful API serves as the backend foundation for the FlixHive client applications developed in later projects.

Goal / Purpose

The goal of this project was to build a fully functional REST API that demonstrates backend development skills, database design, and user authentication using the MERN stack. It emphasizes clean architecture, secure data handling, and the ability to connect a non-relational database to client-side applications for dynamic data interaction.

Challenges & Solutions

Challenge: Implementing secure user authentication and authorization while maintaining API performance and data integrity.

Solution: Integrated Passport.js with JWT tokens for authentication and implemented bcrypt for password hashing, ensuring robust security throughout the application.

Challenge: Securely connecting to MongoDB Atlas without exposing credentials in the codebase.

Solution: Implemented environment variables using "process.env.CONNECTION_URI" to store database credentials securely, ensuring seamless deployment across different environments.

Challenge: Ensuring data integrity and providing meaningful error responses to API consumers.

Solution: Implemented comprehensive input validation using express-validator with custom validation rules (e.g., alphanumeric usernames, email format validation) and structured error handling middleware.
