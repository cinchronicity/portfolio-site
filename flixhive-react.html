Project Overview

The FlixHive app is the client-side interface for a full-stack movie application built with React, connecting to the FlixHive API. It provides users with a responsive, single-page experience to explore movies, view detailed information, and manage their personal profile and favorite movie list. The app emphasizes modern front-end principles like component-based architecture, state management, and routing for seamless navigation.

Goal / Purpose

The goal of this project was to build the client-side of a full-stack application using the MERN (MongoDB, Express, React, Node.js) stack. It demonstrates proficiency in building dynamic, data-driven user interfaces that consume RESTful APIs while maintaining clean, maintainable code and a smooth, user-focused design

Challenges & Solutions

Challenge: Managing complex application state and implementing smooth routing between different views and components.

Solution: Utilized React Router for seamless navigation and implemented proper state management patterns with React hooks for efficient data flow.

Challenge: Seamless integration with backend API for authentication, movie data, and user preferences

Solution: Utilized Axios for HTTP requests with proper error handling, token-based authentication, and loading states

Challenge: Building reusable, maintainable components with proper data flow

Solution: Created modular component structure with PropTypes validation, ensuring type safety and component reusability

Lessons Learned

  • Mastered React component lifecycle and state management with hooks
  • Gained experience with React Router for single-page application navigation
  • Learned to integrate frontend applications with REST APIs using Axios
  • Improved understanding of responsive design with React Bootstrap
  • Practiced user authentication flow and token management in React

Future Improvements
